DPG visual harness

Manual, browser-based validation of @francav/bpmn-js-adapter (canvas overlays + selection) and dpg-modeler (live governance modeler). See the README in this package for the checklist of what to look for.

Section A — bpmn-js-adapter (read-only canvas decoration)

A read-only NavigatedViewer with the sample compiled once via compileModelInBrowser, mapped through mapCompilerResult, and painted onto the canvas with DpgCanvasBinding.apply() (Axis-Y rings, Axis-X badges, finding markers). The buttons drive DpgCanvasSelection.focusElement(); canvas selections are logged below. 

(canvas selection log)

Section B — dpg-modeler (live governance modeler)

An editable Modeler driven by startReferenceModeler with the real createCompilerClassifier(). The consolidated inspector renders to the right; analysis runs on load, every edit re-classifies, clicking a finding/matrix dot selects on the canvas, and changing the profile/policy dropdown re-runs the analysis.
